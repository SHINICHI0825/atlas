Atlas Design System

Atlas全体のUIを統一し、今後どの画面を追加してもブランド品質を維持するための共通言語です。ブランドガイドライン（ロゴ・タイポグラフィ・マーク）とは独立した、プロダクトUIのためのトークンとコンポーネント仕様集です。

Minimal Token-driven Apple / Stripe / Linear / Notion / Vercel 参考

03

Spacing

8pxグリッドを基準とし、4pxのハーフステップのみ許可します。すべての padding / gap / margin はこのスケールから選択します。

04

Border Radius

角丸は要素の階層と大きさに応じて6段階。ピル型（full）はバッジ・アバターのみに使用します。

05

Shadow

影は最小限に留め、階層の区別のみに使用します（装飾目的の強い影は使用しません）。

06

Button

Primary は1画面につき1箇所を目安に。Secondary / Ghost は併用可能。Danger は破壊的操作専用。

Variant用途Height

.btn-danger削除・破棄など破壊的操作38px

Input

ラベル・フィールド・ヘルプテキストを1セットとして扱います。エラー時はボーダーとヘルプテキストの色のみ変化させます。

Card

白背景・1pxボーダー・shadow-xsを基本形とします。クリック可能なカードのみhoverでshadow-mdへ遷移します。

標準カード

border + shadow-xs。情報のグルーピングに使用する最も基本的な単位です。

インタラクティブカード

hoverで浮き上がり、クリック可能であることを示します。

アクセントカード

上部にPrimaryのアクセントラインを添えた強調バリエーション。

12

Modal

オーバーレイは neutral-900 の50%。フッターのアクションは常に右寄せ、Primaryを右端に配置します。

在庫データを削除しますか？

この操作は取り消せません。アップロード済みの在庫データがすべて削除されます。
13

Empty State

アイコン・見出し・説明・任意のCTAの4段構成。否定的な表現を避け、次のアクションを提示します。

データがまだありません

CSVをアップロードすると、ここに分析結果が表示されます。

該当する結果がありません

フィルター条件を変更して再検索してください。

14

Loading

初回読込はSkeleton、部分更新やボタン内の待機はSpinner、進捗が明確な処理はProgress Barを使用します。

15

Chart Color

系列順にPrimary → Cyan → 補助色の順で割り当てます。ステータス色（success/warning/danger）はチャートの系列色としては使用せず、意味づけが必要な場合のみ使用します。

16

Icon Size

Line icon（stroke-width 2）で統一。ボタン内はsm、単体アクションはmd、空状態やモーダル見出しはlg〜xlを使用します。

17
